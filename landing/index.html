What it does

Most migration tools move your data and drop everything else. SunSetter carries over your schema logic too.

Constraint Translation

SQL CHECK constraints are parsed and converted to Convex validators. CHECK (age >= 18 AND age <= 120) becomes v.number().gte(18).lte(120).

Index Suggestions

Analyzes your foreign keys and column usage to suggest Convex indexes. Each suggestion includes the field, a priority level, and copy-paste-ready code.

Full Code Generation

Generates schema.ts, typed queries, mutations, actions, HTTP endpoints, TypeScript interfaces, and React hooks for every table.

Streaming Migration

Moves data with cursor-based pagination so memory stays constant regardless of table size. Supports parallel table migration, checkpointing, and rollback.

PostgreSQL Extras

Auto-detects pg_enum types and maps them to v.union(). Handles partitioned tables, materialized views, and extension types like ltree and hstore.

Production Tooling

Dry-run mode, Slack notifications on migration status, PII data masking for GDPR, post-migration row-count verification, and an MCP server for Claude integration.

What you get

Point SunSetter at a SQL database and it writes your Convex backend for you.

convex/schema.ts
export default defineSchema({
  users: defineTable({
    name: v.string(),
    email: v.string(),
    age: v.number().gte(18).lte(120),
    role: v.union(
      v.literal("admin"),
      v.literal("editor"),
      v.literal("viewer")
    ),
    org_id: v.id("organizations"),
  })
    .index("by_email", ["email"])
    .index("by_org", ["org_id"])
    .searchIndex("search_name", {
      searchField: "name",
    }),
});
convex/users.ts
export const list = query({
  args: {
    paginationOpts: paginationOptsValidator,
  },
  handler: async (ctx, args) => {
    return ctx.db
      .query("users")
      .paginate(args.paginationOpts);
  },
});

export const create = mutation({
  args: {
    name: v.string(),
    email: v.string(),
    age: v.number(),
    role: v.string(),
    org_id: v.id("organizations"),
  },
  handler: async (ctx, args) => {
    return ctx.db.insert("users", args);
  },
});

sunsetter migrate --mode schema-and-data Full migration with data transfer
sunsetter introspect Inspect source schema without writing files
sunsetter preflight Validate everything before migrating
PostgreSQL Deep Support

Enum Detection

Queries pg_enum directly and maps each value to v.union(v.literal(...)). No manual mapping. Works with custom types and composite enums.

Partition & View Handling

Detects RANGE/LIST/HASH partitioned tables and merges them into single Convex collections. Materialized views are marked read-only with a warning about refresh logic.

Extension Types

Maps ltree, hstore, cube, cidr/inet, isbn, and other extension types to appropriate Convex validators.
